# Design System Documentation

## Overview
The Resume‑AI design system provides a consistent visual language built on **Tailwind CSS** and a set of custom design tokens. All UI components (`Button`, `FormField`, `Input`, `Textarea`, `Card`, `ToastNotification`, `Skeleton`, etc.) consume these tokens to ensure consistent spacing, typography, color, and interaction patterns.

---

## Design Tokens

### Spacing
- 4 px base unit (`--spacing-0` … `--spacing-12`).
- Use Tailwind spacing utilities (`p-1`, `gap-2`) which map directly to the tokens.

### Typography
- Semantic classes: `heading-primary`, `heading-secondary`, `body-large`, `body-medium`, `body-small`.
- Font‑size scale is built on the 4 px spacing grid.

### Color Palette
| Role | Token | Example Usage |
|------|-------|---------------|
| Primary | `--color-primary` | `bg-[--color-primary] text-[--text-primary]` |
| Secondary | `--color-secondary` | `bg-[--color-secondary]` |
| Success | `--color-success` | `text-[--color-success]` |
| Warning | `--color-warning` | `bg-[--color-warning]/10` |
| Danger | `--color-danger` | `border-[--color-danger]` |
| Neutral | `--color-neutral` | `text-[--color-neutral]` |

### Elevation
- Tokens: `--elevation-none`, `--elevation-low`, `--elevation-medium`, `--elevation-high`, `--elevation-dialog`.
- Apply via `shadow-[--elevation-medium]` or the component’s built‑in elevation prop.

## Component Usage Guidelines

### Button (`src/components/ui/Button.tsx`)
- **Variants:** `primary`, `secondary`, `ghost`, `destructive`, `outline`, `link`.
- **Sizes:** `sm`, `md` (default), `lg`.
- **Props:** `loading`, `leftIcon`, `rightIcon`, `variant`, `size`.
- **Accessibility:** Focus ring (`focus-visible:ring-2`) and minimum hit area (`min‑h‑[44px]`).

### FormField (`src/components/ui/FormField.tsx`)
- Wraps a label, optional help text, and the input component.
- Always associate `label` with input via `htmlFor`/`id`.
- Provides error state styling (`border‑danger`).

### Input & Textarea (`src/components/ui/Input.tsx`, `src/components/ui/Textarea.tsx`)
- Accept optional `leftIcon`/`rightIcon`.
- Consistent focus ring and hover background.
- Minimum touch target handled by parent `FormField`.

### Card (`src/components/ui/Card.tsx`)
- Accepts `elevation` prop (`none`, `low`, `medium`, `high`, `dialog`).
- Use for containers that need visual separation.

### ToastNotification (`src/components/ui/ToastNotification.tsx`)
- Use via `ToastProvider` context.
- Variants: `success`, `error`, `warning`, `info`.
- Auto‑dismiss after 4 s unless `persistent`.

### Skeleton (`src/components/ui/Skeleton.tsx`)
- Use for loading placeholders. Pass `variant` (`text`, `avatar`, `button`, `image`) or custom `className` for size.

## Interaction Patterns
- **Hover / Focus:** All interactive elements use Tailwind’s `transition` utilities and a `focus-visible` ring for accessibility.
- **Micro‑interactions:** Buttons have a press ripple (primary variant) and scale‑down animation on active state.
- **Reduced Motion:** Users who prefer reduced motion are respected via the `motion-reduce` media query – animations are disabled.

---

## Testing Guidelines
- **Unit Tests:** Use **Vitest** with **React Testing Library** for component behavior.
- **Integration Tests:** Combine components in a realistic page hierarchy.
- **Visual Regression:** Use **Playwright** with screenshot comparisons (`e2e/visual.test.ts`).
- **Accessibility:** Run **axe** via `jest-axe` or `@testing-library/react` to assert no violations.
